t1 | how to define/specify "tab" on command line

Author: Arwin Lashawn Date: April 9, 2020  16:16:38 Category: tutorials

Note

The "cut" command specifies tab space as the delimiter by default, hence making it very powerful for .tsv files. Just using it as an example!

Important: One time I was working with a .tsv files but defining "tab" as the delimiter didn't work as expected. This was because although the file has a .tsv extension, the separators in the file also included "|", and it appeared before tab spaces. For some context, I was working with BLAST results. A reminder to "less" the file first to get a sense of the delimiter used!

Two ways to define "tab”

Examples below are for grabbing the third field of a .tsv file:

  1. On the command line, Ctrl + V followed by Tab. Worked for me on both Linux and Git Bash command lines.

  2. Use $'\t'.

    You may be wondering, are the dollar sign and single quotes actually necessary? Yes.

    Explanation: If you go ahead and just specify '\t', you will get an error saying "the delimiter must be a single character". By adding a dollar sign to it, you get a general form of $'string' which is treated uniquely. The result? The word expands to a string, and the backslash-escaped character, in this case "\t", is replaced by a "horizontal tab" according to the ANSI-C standard.
